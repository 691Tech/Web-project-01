‚ú® Key Features Implemented
The application is built to simulate a full user journey within the logistics domain, featuring the following pages and functionalities:
Multi-Role Sign-Up Flow (Landing Page):
A dedicated landing view is present to onboard users based on their roles, which include Customer/Shipper, Transporter/Booking Agent, Truck Operator/Fleet Owner, and Broker/Truck Supplier. Each role is presented with a distinct and appealing card for selection.
Detailed Registration Modals:
The prototype includes four dedicated registration forms (modals) for each user type. These forms collect all necessary basic, contact, and company details as per the provided wireframes. It also includes validation placeholders for critical information such as mobile number, email, GST number, and PAN number.
Sign In & Authentication Simulation:
The Sign In view is fully functional, accepting Mobile Number and Password. A Forgot Password link is included, which triggers a Reset Password Modal. Additionally, a simulated QR Code Login section provides a modern login alternative. The Functional Login/Logout feature successfully simulates authentication, redirecting the user to the dashboard upon "Sign In" and back to the login page upon "Logout."
Dashboard Simulation:
A comprehensive post-login view (/dashboard) is provided. This dashboard showcases key performance indicators (KPIs) like Active Loads, Completed Shipments, and Pending Payments, demonstrating a clear, professional layout suitable for daily logistics operations.
Full Responsiveness:
The entire application design is fully adaptive, ensuring a seamless user experience across all devices, including mobile phones, tablets, and desktops.
